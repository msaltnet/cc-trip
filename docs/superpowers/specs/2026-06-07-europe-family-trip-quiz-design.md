# 유럽 가족 여행 준비 Quiz 사이트 — 설계 문서

작성일: 2026-06-07

## 1. 개요

가족의 유럽 여행 준비를 돕고 기대감을 높이기 위한 **모바일 친화 정적 웹사이트**.
카테고리별 학습 본문을 읽고, 챕터 단위로 랜덤 퀴즈를 풀며 복습한다.

**핵심 가치**
- **프라이버시:** 서버·DB 없이 클라이언트 사이드에서만 동작. 가족 개인정보를 외부로 보내지 않는다.
- **집중도:** 제한 시간(10분)과 랜덤 문제 은행으로 실제 시험 같은 몰입감 제공.
- **확장성:** 카테고리·챕터·문제가 **코드 수정 없이** 데이터 파일만으로 계속 추가되는 구조.

## 2. 기술 스택

- **순수 HTML/CSS/JS (Vanilla)** — 빌드 도구·외부 의존성 0.
- 멀티페이지 구조(페이지별 HTML). Vanilla에서 라우팅 코드 없이 가장 단순하고, 각 파일이 한 가지 역할만 갖는다.
- 시스템 폰트 스택, 외부 폰트/CSS 프레임워크 없음 → 오프라인 동작 가능.
- 배포: 폴더 그대로 GitHub Pages 등 정적 호스팅에 업로드.

## 3. 정보 구조 (3계층 드릴다운)

```
카테고리 (Category)
  └─ 챕터 (Chapter)
       ├─ 본문 (sections)
       └─ 문제 (questions)
```

- 카테고리: 주제 대분류 (예: 생존 영어 회화).
- 챕터: 카테고리 내 학습 단위 (예: 공항 입국 심사). 영어회화는 ~30챕터 규모를 상정.
- 챕터마다 학습 본문과 문제를 함께 보유. 퀴즈는 **챕터 단위**로 출제.

## 4. 파일 구조

```

├── result.html             # 결과·해설 화면
```

## 5. 화면 흐름

```
index.html
  ↓ 카테고리 클릭
category.html?cat=survival-english       # 챕터 목록 (스크롤 리스트)
  ↓ 챕터 클릭
chapter.html?cat=survival-english&ch=ch01  # 본문 + [퀴즈 시작하기]
  ↓ 퀴즈 시작 → PIN 모달 (통과 시)
quiz.html?cat=survival-english&ch=ch01     # 랜덤 10문항 + 상단 sticky 타이머
  ↓ 제출 or 시간만료 자동제출
result.html                                # 점수 + 전체 문항 해설
```

- 화면 간 데이터 전달은 **sessionStorage** 사용 (탭 닫으면 사라지는 일시 전달용. 영구 저장 아님).
- URL 파라미터: `cat`(카테고리 id), `ch`(챕터 id).

## 6. 데이터 스키마

### 6.1 `data/categories.json`

```json
[
  {
    "id": "survival-english",
    "icon": "🗣️",
    "title": "생존 영어 회화",
    "summary": "공항·식당·길찾기 등 상황별 표현",
    "file": "survival-english.json"
  }
]
```

### 6.2 `data/<category>.json`

```json
{
  "id": "survival-english",
  "title": "생존 영어 회화",
  "chapters": [
    {
      "id": "ch01",
      "title": "공항 입국 심사",
      "summary": "여권·입국 목적·체류 기간 표현",
      "sections": [
        { "heading": "입국 목적 말하기", "body": "Tourism. / I'm here for sightseeing. 라고 답하면 됩니다." }
      ],
      "questions": [
        {
          "id": "se_ch01_001",
          "type": "multiple_choice",
          "question": "입국 심사관이 방문 목적을 물을 때 가장 적절한 답은?",
          "options": ["Tourism.", "I don't know.", "Business class."],
          "answerIndex": 0,
          "explanation": "방문 목적은 Tourism 또는 Sightseeing으로 답합니다."
        }
      ]
    }
  ]
}
```

**규칙**
- `answerIndex`는 0-based. `options` 길이는 가변(3~4지선다 자유).
- `type`은 현재 `multiple_choice`만 지원. 향후 OX·주관식 확장을 위해 필드로 유지.
- 문제 `id` 접두사로 출처 추적: `se_ch01_001` = 카테고리약어_챕터_번호.

**확장 방법 (코드 수정 0)**
- 카테고리 추가: `data/<새파일>.json` 1개 + `categories.json`에 한 줄 등록.
- 챕터 추가: 해당 카테고리 파일의 `chapters` 배열에 항목 추가.
- 문제 추가: 해당 챕터의 `questions` 배열에 항목 추가.

## 7. 핵심 기능 요구사항

### 7.1 PIN 인증 모달
- `chapter.html`에서 `[퀴즈 시작하기]` 클릭 → PIN 입력 모달 팝업.
- `config.js`의 `QUIZ_PIN` 상수와 대조. 일치 시 sessionStorage에 통과 플래그 기록 후 `quiz.html` 이동.
- 불일치 시 모달 내 경고 메시지 노출, 진입 차단.
- **보안 주의:** 클라이언트 사이드이므로 PIN은 소스코드에 노출된다. 실제 보안이 아니라 "장난으로 들어오지 않게 하는 가벼운 관문" 수준임을 README/문서에 명시.

### 7.2 랜덤 10문항 출제
- 해당 챕터 `questions`를 Fisher-Yates 셔플 후 앞에서 `min(10, 전체개수)` 추출.
- 문제 부족 시(10개 미만) **있는 만큼만** 출제. 점수는 `맞힌수 / 출제수`로 표시, 타이머는 그대로 10분.
- 출제 문항 + 사용자 답을 sessionStorage에 보관(화면 간 전달).
- 한 페이지 세로 스크롤로 전 문항 표시.

### 7.3 타이머 & 자동 제출
- 상단 `position: sticky` 영역에 `MM:SS` 카운트다운(600초).
- 남은 시간 60초 이하 시 색상 강조(빨강).
- 0초 도달 → 경고(alert) → 현재까지 선택된 답 기준 즉시 자동 제출 → `result.html`.

### 7.4 채점 & 해설
- 점수: `맞힌 수 / 출제 수` (예: 7/10).
- 해설 리스트: 출제된 전 문항을 순서대로 — 문제, 사용자 선택, 정답, O/X 표시, `explanation`.
- O는 초록, X는 빨강으로 정오답 시각 구분.
- `[다시 풀기]`(같은 챕터 새 랜덤) / `[목차로]` 버튼.

## 8. UX & 스타일

- 모바일 우선. 가족 여행 설렘이 느껴지는 밝고 친근한 톤, 카테고리별 이모지 아이콘.
- 단일 `css/style.css`, CSS 변수로 색상·간격 토큰화.
- 카드형 리스트, 큰 터치 타깃, 부모님도 보기 편한 가독성 높은 폰트 크기.
- 미선택 문항이 있으면 제출 시 "안 푼 문항이 N개 있습니다. 제출할까요?" 확인.

## 9. 에러 처리

| 상황 | 처리 |
|------|------|
| 잘못된 URL 파라미터(없는 cat/ch) | "찾을 수 없는 항목" 안내 + 목차 이동 버튼 |
| JSON 로드 실패(fetch 에러) | "콘텐츠를 불러오지 못했습니다. 새로고침 해주세요" 안내 |
| 퀴즈 화면 직접 진입(PIN 통과 기록 없음) | chapter로 되돌림 (PIN 우회 방지) |
| 챕터 문제 0개 | 퀴즈 시작 버튼 비활성화 + "준비 중인 챕터" 표시 |

## 10. 테스트 전략

- 핵심 순수 함수(`shuffle`, `score`, `formatTime`, `getParam`)를 `js/util.js`에 분리해 부수효과 없이 작성.
- 자동화 테스트 프레임워크는 도입하지 않음(무의존성 원칙 유지). 대신 TRD에 **수동 검증 체크리스트** 포함.
- 로직 함수를 순수하게 유지해 추후 테스트 추가 여지를 남긴다.

## 11. 기본 카테고리 (5종)

| icon | 카테고리 | 내용 요약 |
|------|----------|-----------|
| 🗣️ | 생존 영어 회화 | 공항·식당·길찾기 등 상황별 표현 (최대 ~30챕터) |
| 🏙️ | 방문 도시 상식 | 도시별 명소·교통·주의사항 |
| 🤝 | 문화·에티켓 | 팁 문화, 인사, 식사 예절 |
| 🎒 | 짐 싸기 체크리스트 | 소매치기 방지 용품, 어댑터, 비상약, 운동화 |
| 💶 | 환전·결제 상식 | 트래블카드, DCC 차단, 택스리펀 |

> PRD에 언급된 "위급 상황·안전 수칙" 카테고리는 동일 구조로 파일만 추가하면 되므로, 기본 5종으로 시작하고 추후 확장한다.
