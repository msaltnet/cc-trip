# 여행 상식 카테고리 통합 설계

작성일: 2026-06-10

## 배경

현재 카테고리 중 두 개가 콘텐츠가 얇다.

- **짐 싸기 체크리스트**(`packing`) — 챕터 1개 (`ch01 필수 준비물`)
- **환전·결제 상식**(`finance`) — 챕터 2개 (`ch01 카드 결제`, `ch02 택스리펀`)

메인 화면에 카테고리 카드로 노출되는데 알맹이가 적은 카테고리가 둘이나 있다. 이 둘을 **"여행 상식"** 하나로 합쳐 각각을 챕터로 넣으면 카테고리 무게가 균형이 맞고, 실무·준비 노하우 주제를 챕터로 계속 확장하기 좋다.

`culture`(문화·에티켓)가 인사·식사·종교·쇼핑·시간감각·안전(소매치기)·매너를 이미 폭넓게 다루므로, "여행 상식"은 **실무·준비 노하우** 쪽으로 차별화해 주제 중복을 피한다.

## 목표

1. `packing` + `finance`를 단일 카테고리 **"여행 상식"**(`travel-tips`)으로 통합한다.
2. 기존 3개 챕터(짐 싸기, 카드 결제, 택스리펀)는 본문·세트를 그대로 이전한다.
3. 신규 4개 챕터(데이터·eSIM, 여행자보험, 비행기·수하물, 숙소 체크인·환전)를 추가한다.
4. 코드 수정 없이 `data/` JSON 변경만으로 완료한다(앱은 완전 데이터 주도, 카테고리 id 하드코딩 없음 — 검증 완료).

## 비목표

- 기존/신규 챕터를 권장 기준인 10세트까지 채우지 않는다. 기존 분량(챕터당 2세트)에 맞춘다.
- `culture`·`survival-english`·`city-info` 등 다른 카테고리는 손대지 않는다.
- 앱 코드(`js/`, `*.html`)·빌드 스크립트는 변경하지 않는다.

## 변경 사항

### 1. 카테고리

- **신규 파일** `data/travel-tips.json`
  - `id: "travel-tips"`, `icon: "🧳"`, `title: "여행 상식"`
  - `summary: "짐 싸기, eSIM, 수하물, 숙소·환전, 카드 결제, 택스리펀"`
- `data/categories.json`
  - `packing`·`finance` 항목 제거
  - `travel-tips` 항목 추가 (`culture` 다음 위치)
- **삭제** `data/packing.json`, `data/finance.json`

### 2. 챕터 구성 (여행 흐름 순서: 준비 → 이동 → 현지 → 귀국)

| ch   | 제목                 | 아이콘 | 출처            |
|------|----------------------|--------|-----------------|
| ch01 | 짐 싸기              | 🎒     | packing ch01 이전 |
| ch02 | 데이터·eSIM·로밍     | 📱     | 신규            |
| ch03 | 여행자보험           | 🛡️     | 신규            |
| ch04 | 비행기·수하물 규정   | ✈️     | 신규            |
| ch05 | 숙소 체크인·환전     | 🏨     | 신규            |
| ch06 | 카드 결제 요령       | 💳     | finance ch01 이전 |
| ch07 | 택스리펀             | 🧾     | finance ch02 이전 |

> 아이콘은 챕터 `title` 텍스트에 직접 넣지 않고, 기존 챕터 표기 방식을 따른다(현재 챕터 제목에 이모지를 쓰지 않으므로 본 통합에서도 제목은 텍스트만 유지하고 이모지는 설계 문서상 식별용으로만 사용).

### 3. 콘텐츠 분량

- **이전 3챕터**(ch01 짐싸기, ch06 카드결제, ch07 택스리펀)
  - 본문 `sections`와 기존 세트 2개(각 10문항)를 그대로 이전.
  - 문제 `id`만 새 접두사 `tt_chNN_setNN_NNN` 형식으로 재명명(출처 추적 일관성).
- **신규 4챕터**(ch02~ch05)
  - 본문 `sections` 작성.
  - 세트 **2개**, 각 **10문항**(`multiple_choice` 또는 `ox`) 구성해 바로 플레이 가능하게.
  - 같은 챕터의 두 세트는 같은 주제·개념을 다루되 지문·보기·순서를 다르게 구성(README 규칙 3).

#### 신규 챕터 본문 주제 가이드

- **ch02 데이터·eSIM·로밍**: eSIM/유심/로밍 비교, 설정 절차, 데이터 절약 팁.
- **ch03 여행자보험**: 보장 항목, 의료·도난 사고 대처, 청구용 서류·영수증 보관.
- **ch04 비행기·수하물 규정**: 기내/위탁 수하물 무게·액체(100ml) 규정, 보조배터리 기내 휴대, 환승·도착 대응.
- **ch05 숙소 체크인·환전**: 호텔/에어비앤비 체크인, 시티택스·보증금, 현금 환전 요령(공항 환전 비추천, 현지 ATM).

### 4. 검증

- 통합 후 `node`로 `data/travel-tips.json` 파싱 및 챕터/세트/문항 수 확인(7챕터, 각 2세트, 세트당 10문항).
- `data/categories.json`이 유효 JSON이며 `travel-tips` 1건만 추가·`packing`/`finance` 제거됐는지 확인.
- 로컬 HTTP 서버(`python -m http.server`)에서 흐름 점검: 메인 → 여행 상식 → 각 챕터 본문 → 퀴즈 시작 → 결과.

## 문서 갱신

- `README.md`의 카테고리 추가 예시 등 본문에 `packing`/`finance`를 직접 가리키는 서술이 있으면 갱신(현재 예시는 가상 `safety`라 영향 없음 — 확인 후 필요 시 수정).

## 리스크 / 주의

- 문제 `id` 재명명은 출처 추적용 권장 규칙이며 앱 동작에 영향 없음(앱은 `id`로 분기하지 않음).
- 삭제되는 `packing.json`/`finance.json`을 참조하는 외부 북마크/링크가 있다면 깨질 수 있으나, 가족용 내부 사이트라 영향은 무시 가능.
