# 여행 상식 재구성 설계 (7챕터 → 3챕터)

작성일: 2026-06-10

## 배경

`data/travel-tips.json`("여행 상식")은 현재 7챕터 × 2세트(총 14세트, 140문항)다. 두 가지 문제가 있다.

1. **본문이 너무 얇다.** 챕터당 본문(sections)은 핵심을 한 줄씩 요약한 2~4문장뿐인데 문제는 챕터당 20개라, 본문을 읽고도 풀기 어려운 문제가 있다. 답변 가능성 감사 결과 확인된 갭:
   - **하드 갭** — ch01(짐싸기) `set01_005`/`set02_005`/`set02_009`: 본문은 OTC 상비약만 나열하고 처방약을 전혀 언급하지 않는데, 정답은 "타인의 처방약은 부적절"을 알아야 풀린다.
   - **추론 의존** — ch06(카드결제) `set02_006`(대형 백화점 카드 가능)은 소거법으로만 가능, `set01_009`/`set02_008`("자동 현금 환급" 오답)은 본문 근거 없음; ch07(택스리펀) `set01_010`/`set02_009`("손가방에 휴대")는 "부치기 전 처리"의 해석; ch05 `set02_010`("키박스가 가장 많이 쓰임")은 본문이 단정하지 않음.
2. **챕터당 세트가 2개뿐**이라 한 챕터의 분량이 빈약하다.

## 목표

1. 7챕터를 **주제별 3챕터**로 통합한다.
2. 각 챕터를 **5세트**로 구성한다(챕터당 50문항, 총 150문항). "챕터당 5세트 이상" 요건 충족.
3. 각 세트를 **챕터 전체 주제를 아우르는 테마 통합 세트**로 구성한다(소주제 혼합).
4. 본문을 보강하고 문제를 손질해 **모든 문제의 정답 근거가 본문에 담기도록** 한다.

## 비목표

- 권장 상한인 챕터당 10세트까지 채우지 않는다(5세트로 통일).
- 다른 카테고리(survival-english, city-info, culture)는 손대지 않는다.
- 앱 코드(`js/`, `*.html`)·빌드 스크립트는 변경하지 않는다(데이터 주도, 카테고리/챕터 id 하드코딩 없음 — 확인됨).

## 챕터 구성

| ch   | title              | 통합 소주제                                              | summary 초안 |
|------|--------------------|----------------------------------------------------------|--------------|
| ch01 | 출발 전 준비       | 짐 싸기 · 데이터/eSIM/로밍 · 여행자보험                  | 짐 싸기, eSIM·로밍, 여행자보험 |
| ch02 | 환전·결제·세금환급 | 현금 환전 · 카드 결제(DCC·트래블카드) · 택스리펀         | 현금 환전, 카드 결제, 택스리펀 |
| ch03 | 비행기·수하물·숙소 | 기내/위탁 수하물·보조배터리·환승 · 숙소 체크인·시티택스·보증금 | 수하물 규정, 숙소 체크인 |

> 기존 ch05 "숙소 체크인·환전"은 분리: **현금 환전 요령** → ch02, **체크인·시티택스·보증금** → ch03.

카테고리 메타(`categories.json`)는 `id`/`file`/`icon`/`title` 그대로 두고, `summary`만 새 구성에 맞게 갱신한다(예: "출발 준비, 환전·결제, 수하물·숙소").

## 본문(sections) 설계

각 챕터 본문은 소주제별 섹션들로 구성하되, 기존 본문보다 충실히 다시 써서 **그 챕터 문제들이 묻는 사실을 모두 포함**한다. 섹션 수는 소주제 합(챕터당 6~9개) 정도.

### ch01 출발 전 준비 — 다뤄야 할 사실
- 짐싸기: 소매치기 방지 용품(시큐리티 와이어·다이얼 자물쇠·목걸이형 안전지갑), C타입 콘센트·멀티 어댑터·USB 멀티 충전기, **상비약은 본인용 OTC(소화제·진통제·지사제·밴드·멀미약) 위주이고 타인의 처방약은 부적절**(하드 갭 해소), 길든 편한 운동화.
- eSIM·로밍: eSIM(QR 즉시 개통·카드 교체 불필요), 데이터 유심(카드 교체·한국 번호 수신 제한 가능), 통신사 로밍(편하나 비쌈), 단기 가족여행엔 eSIM/유심 가성비, '데이터 로밍' 켜기, 듀얼(유심+eSIM) 병행, 와이파이 활용, 오프라인 지도·번역 다운로드, 백그라운드/자동업데이트 끄기.
- 보험: 보장 항목(의료비·휴대품 도난파손·항공지연·수하물분실·배상책임), 가족 단위 가입·상품마다 한도 다름, 진단서·영수증 보관, 도난 시 폴리스 리포트, 귀국 후 청구, 원본 보관+사진 백업.

### ch02 환전·결제·세금환급 — 다뤄야 할 사실
- 현금 환전: 공항 환전소 환율 나쁨·소액만, 현지 ATM 현지통화 인출 유리, ATM에서도 원화환산(DCC) 거절.
- 카드 결제: DCC(KRW 선택 시 이중 수수료) 거절·EUR 선택, 트래블카드 충전·수수료 적음·분실 시 앱 즉시 정지, **카드가 잘 되는 곳(대형 상점·백화점·체인점)과 현금이 필요한 곳(재래시장·공중화장실·소규모 상점)을 대비**(추론 의존 해소), 선불카드는 결제 시 자동 현금 환급 같은 기능은 없음(오답 근거 명시).
- 택스리펀: EU 비거주자·일정 금액 이상 시 VAT 일부 환급, 상점에서 Tax Free 서류, 공항에서 ①세관 도장 →②환급 창구/키오스크 순서, **환급 대상 물건·서류는 부치지 말고 기내(손가방)에 휴대**(추론 의존 해소).

### ch03 비행기·수하물·숙소 — 다뤄야 할 사실
- 수하물: 기내 액체 용기당 100ml·1L 투명 지퍼백 1개·큰 액체는 위탁, 위탁 무게·개수 항공사·요금제마다 다름·출발 전 확인, LCC 기내수하물도 엄격·초과 시 추가요금, 보조배터리·리튬배터리 위탁 금지·기내 휴대·용량 제한(보통 100Wh, 약 27,000mAh), 환승 연결시간 여유·위탁 자동연결 확인, 도착 후 수하물 수취대(baggage claim).
- 숙소: 체크인 가능 시간 미리 확인·여권 제시, 에어비앤비는 호스트와 체크인 방법(직접 만남·셀프 체크인·키박스 등) 사전 조율, 시티택스(숙박세) 1박당 현장 부과, 보증금(디포짓) 카드 임시 홀드·체크아웃 시 해제.

## 세트(sets) 설계

- 각 챕터 **5세트**(`set01`~`set05`), 세트당 **정확히 10문항**.
- 각 세트는 챕터의 **모든 소주제를 고르게 섞는다**. 예: ch01 한 세트 = 짐싸기 3~4 + eSIM 3~4 + 보험 2~3문항. 5세트에 걸쳐 각 소주제가 균형 있게 반복되도록.
- 세트끼리는 같은 사실을 다루되 지문·보기·순서를 다르게(암기 방지). 세트 간 동일 지문 복제 금지.
- 유형은 `multiple_choice`(보기 4개 권장, 0-based `answerIndex`)와 `ox`(보기 없음, 0=O/참, 1=X/거짓)를 섞는다(세트당 mc ~7 + ox ~3).
- 문제 `id`: `tt_chNN_setNN_NNN`(새 챕터 번호 ch01~ch03 기준 재부여).
- 모든 문제에 비어있지 않은 한국어 `explanation`.

### 문제 출처 (재활용 + 신규)
기존 140문항을 최대한 재활용하되, 새 챕터 주제 풀로 재배치하고 부족분만 신규 작성한다. 손질·재작성 시 위 "본문 설계"의 보강 내용을 반영해 모든 정답이 본문으로 풀리게 한다.

- **ch01 풀**: 기존 짐싸기 20 + eSIM 20 + 보험 20 = 60문항에서 50문항 선별·재배치. 하드 갭 3문항(처방약)은 본문 보강으로 해소되며 그대로 활용 가능.
- **ch02 풀**: 기존 카드결제 20 + 택스리펀 20 + (기존 ch05의 환전 관련 ~8문항) ≈ 48 → 50 맞추도록 소량 신규. `set02_006`(백화점)·"자동 현금 환급" 오답·"손가방" 문항은 본문 보강에 맞춰 손질.
- **ch03 풀**: 기존 수하물 20 + (기존 ch05의 체크인 관련 ~12문항) ≈ 32 → 50 맞추도록 ~18문항 신규(숙소·수하물 주제 내에서).

## 적용 / 마이그레이션

- `data/travel-tips.json`을 3챕터(ch01~ch03, 각 5세트) 구조로 재작성한다.
- `data/categories.json`의 `travel-tips` 항목 `summary`만 갱신.
- 다른 파일·코드는 변경하지 않는다.

## 검증

- `node`로 파싱·구조 확인: 3챕터, 각 5세트, 세트당 10문항, 문제 id 150개 전부 고유·`tt_` 접두사.
- 각 문제: `answerIndex` 범위 유효, `ox`는 `options` 없음, `explanation` 존재.
- **답변 가능성 재감사**: 각 챕터 본문 대비 정답 근거 누락 0 (특히 위에서 해소하기로 한 갭들).
- 세트 간 중복(동일 지문) 점검.
- 사이트 전체 무결성(`categories.json`의 모든 파일 로드, 전 문항 유효) 확인.
- 로컬 서버 수동 점검: 메인 → 여행 상식 → 3챕터 → 각 챕터 퀴즈 풀이.

## 리스크 / 주의

- 테마 통합 + 본문 보강은 분량이 크다(150문항). 기존 문항 재활용으로 신규 작성을 최소화하되, 재배치·손질 시 본문 일치와 세트 간 비중복을 우선한다.
- 문제 `id` 재부여로 기존 id 추적성은 끊기지만 앱은 id로 분기하지 않으므로 동작 영향 없음.
- 한 챕터가 여러 소주제를 담으므로, 세트가 한 소주제에 쏠리지 않게 소주제 균형을 명시적으로 관리한다.
