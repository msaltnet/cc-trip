# 문화·에티켓 콘텐츠 확장 설계

- 작성일: 2026-06-10
- 대상 카테고리: `culture` (`data/culture.json`)
- 관련: 생존 영어 회화(`survival-english`) 40챕터를 소재로 활용

## 1. 배경과 목표

`culture` 카테고리는 현재 챕터 1개(팁과 식사 예절)만 있습니다. 영어 회화 콘텐츠를
소재로 삼되, **대화(dialogue)가 아니라 상황·유래·이야기 중심**으로 문화·에티켓
콘텐츠를 확장합니다.

- 영어 회화: "이 상황에서 이렇게 말하세요" (표현 중심)
- 문화·에티켓: "왜 이런 관습이 생겼고, 어떤 상황이며, 어떻게 행동해야 하는가" (이야기 중심)

목적은 가족이 유럽 여행 전 현지 문화를 **이해**하고 기대감을 높이는 것입니다.

## 2. 구조 결정

- **주제별 챕터** 구조로 구성 (회화 40챕터를 1:1 미러링하지 않음).
- 회화 챕터에서 문화적 이야깃거리가 있는 소재를 주제로 재구성.
- 총 **12개 챕터**.
- 각 챕터: 이야기 중심 `sections`(제목별 단락 4~6개) + 퀴즈 `sets` **2~3개**.

기존 포맷(`sections` + `sets`)을 그대로 사용합니다. 신규 코드/렌더러 변경 없음.
`js/chapter.js`는 `dialogues`가 없으면 `sections`를 `<h3>heading</h3><p>body</p>`로
렌더링합니다. 본문은 HTML 이스케이프된 단일 문단이므로, **하나의 이야기를 여러 개의
제목 달린 섹션으로 나눠** 서술합니다(섹션 내 줄바꿈/마크업 미지원).

## 4. 콘텐츠 작성 규칙

### 본문(sections)
- 챕터당 **4~6개 섹션**. 각 섹션 = `{ heading, body }`.
- `heading`: 이야기의 한 장면/소주제 (예: "팁은 원래 '빨리 주세요'였다").
- `body`: 1~3문장의 이야기형 서술. 유래·배경·상황·실천 팁을 자연스럽게 엮음.
- 마지막 섹션은 "실전 팁" 성격으로 행동 지침을 요약(선택).
- 사실 정확성 중시: 나라별 차이가 큰 항목은 "대체로/나라에 따라" 식으로 일반화.

### 퀴즈(sets)
README 콘텐츠 규칙을 따릅니다.
- 1세트 = 정확히 **10문제**.
- 챕터당 **2~3세트**. 세트 간 같은 개념을 다루되 지문·보기·순서를 다르게.
- 문제 유형은 `multiple_choice`(보기 4개) 또는 `ox` 두 가지만.
- `ox`는 `answerIndex` 0=O(참), 1=X(거짓). `options` 미기재.
- `multiple_choice`는 `answerIndex` 0부터.
- 문항 `id` 형식: `culture_chNN_setNN_NNN`.
- 모든 문항에 `explanation` 필수.
- 퀴즈는 본문에서 다룬 **유래·상황·행동 지침**을 묻습니다(회화 표현 암기가 아님).

## 5. 작업 산출물

- `data/culture.json` 전면 재작성 (12챕터, 각 sections + 2~3 sets).
- `data/categories.json`의 `culture.summary`를 확장 내용에 맞게 갱신(선택).
- 코드/렌더러 변경 없음. JSON 검증은 수동(또는 간단한 JSON 파싱 확인).

## 6. 검증

- `culture.json`이 유효한 JSON인지 파싱 확인.
- 각 세트가 정확히 10문제인지, `answerIndex`가 보기 범위 내인지 확인.
- 브라우저에서 카테고리 → 챕터 → 본문 → 퀴즈 → 결과 흐름 점검.

## 7. 범위 밖 (YAGNI)

- 회화처럼 `dialogues`/`keyExpressions` 포맷은 쓰지 않음(이야기 중심이므로).
- 빌드 스크립트(`scripts/`) 추가하지 않음. `culture.json` 직접 편집.
- 새 카테고리·새 화면·새 기능 추가 없음.
